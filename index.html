Data format without the pain.

GOML fixes the frustrations of JSON, YAML, TOML, and XML. Simple syntax. No indentation errors. Comments everywhere.

Why GOML?

Pain points, solved.

#

Comments

Use # or // anywhere. Document your config inline.

# Server config
port = 8080 # production port
{ }

No Indentation Errors

Explicit braces, not fragile indentation. Structure is always clear.

server {
  host = localhost
  port = 8080
}
=

Clean Syntax

Simple key = value. No colons, no quotes on every key.

name = MyApp
port = 8080
active = true
[ ]

Simple Arrays

Comma-separated inline arrays. No dashes, no brackets everywhere.

colors = [red, green, blue]
ports = [80, 443, 8080]
$

References

Reference other values with $path.to.value.

defaults {
  timeout = 30
}
server {
  timeout = $defaults.timeout
}
~

Trailing Commas

Optional trailing commas. No more syntax errors on the last item.

features = [
  auth,
  logging, # trailing comma OK
]
Three Syntax Styles

Choose your style.

Simplest

Flat Style

Best for: Simple configs, .env files

app_name = MyApp
port = 8080
debug = false
database_url = postgres://localhost/myapp
Simpler

Dot Notation

Best for: Partial nesting, Docker Compose

server.host = localhost
server.port = 8080
database.host = db.internal
database.port = 5432
Original

Block Style

Best for: Complex configs, nested structures

server {
  host = localhost
  port = 8080
  ssl {
    enabled = true
  }
}

Mix all three in one file:

# Flat for simple values
name = MyApp
version = 1.0

# Dot notation for partial nesting
server.host = localhost
server.port = 8080

# Block for complex nested structures
database {
  primary {
    host = db-primary.internal
    port = 5432
  }
  replicas = [replica-1.internal, replica-2.internal]
}
